# 30 天学习路线图

## 第 1 周：C 语言基础与学习环境

### 核心学习内容

- C 语言基本语法、变量、运算符、分支和循环
- 函数、数组、字符串的基础使用
- Git 与 GitHub 的基础工作流
- Linux 常用命令入门
- 网络安全学习范围和合法性边界

### 实践成果

- 完成一组 C 语言基础练习
- 建立每日学习日志
- 熟悉本地仓库的目录结构
- 整理常用 Linux 命令笔记

### 验收方式

- 能独立写出基础输入输出、分支、循环和函数程序
- 能说明 Git 的基本提交流程，但不要求熟练
- 第 7 天进行复习，整理本周不会的语法和高频错误

## 第 2 周：指针、内存与线性数据结构

### 核心学习内容

- 指针、数组、字符串和结构体
- 动态内存分配的基础概念
- 线性表、链表、栈、队列
- TCP/IP 与 HTTP 基础概念入门

### 实践成果

- 实现基础链表、栈和队列
- 整理数据结构操作笔记
- 记录 HTTP 请求与响应的基本结构

### 验收方式

- 能解释指针和数组的关系
- 能手写并运行至少两个线性数据结构
- 第 14 天复习数据结构实现中的错误和边界情况

## 第 3 周：C++ 入门、网络基础与合法靶场

### 核心学习内容

- C++ 基础语法、输入输出、函数和类的入门概念
- 树和图的基本概念
- 网络分层、TCP/IP、HTTP 常见字段
- Web 安全基础概念
- CTF 入门题型与公开靶场使用方式

### 实践成果

- 完成 C++ 基础练习
- 整理网络基础笔记
- 完成合法靶场或 CTF 入门题记录

### 验收方式

- 能用自己的话解释 HTTP 请求流程
- 能完成简单 CTF 入门题并写出复盘
- 第 21 天复习编程基础和安全基础之间的薄弱点

## 第 4 周：综合练习与小型项目

### 核心学习内容

- C/C++ 基础巩固
- 常见数据结构复习
- Linux 命令和 Shell 基础练习
- Web 安全与 CTF 入门题复盘
- 小型编程项目整理

### 实践成果

- 完成一个小型编程项目
- 补齐数据结构实现记录
- 整理至少一篇 CTF 或靶场 writeup
- 完成周复盘

### 验收方式

- 项目代码能本地运行
- README 或说明文档能讲清项目用途和运行方式
- 第 28 天进行阶段验收，列出已掌握和仍需补强的内容

## 月末总结

### 核心学习内容

- 回顾 30 天学习记录
- 汇总编程、Linux、网络和 Web 安全的核心收获
- 梳理后续学习方向

### 实践成果

- 完成月度总结
- 整理可展示的项目和笔记
- 规划下一阶段学习重点

### 验收方式

- 检查仓库结构是否清晰
- 检查代码、笔记和 writeup 是否真实可复盘
- 明确下一阶段继续学习的主题
